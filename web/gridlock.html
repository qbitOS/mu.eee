One surface for how things connect

Gridlock is the mapping page: a place to pin how search results, history blocks, layout exports (GridPad), and QPU / geographic context relate—without pretending the wiring is finished. Use it as the narrative home for cross-app throughline while the pipelines land in each app.

Throughline spine (canonical)

One spool ties retrieval, Tonnetz funnel, gamepad/μgrad, QPU, rhythm/video, this hub, and Notes — all AI-capable via isomorphic-export-facet exports and BroadcastChannels (not a separate “magic AI” stack). Full spec + channel table: plans/gridlock-throughline-spine.plan.md (executable spine — not canvas/Three/Mermaid-as-truth; Charm/Go terminal peers; footer terminal = contrail “ghost note”).

  1. Search → user data & snippets; History → context & lineage.
  2. GridPad — fluid AI boilerplate: infer layout, pull content, blocks + JSON (Framer/Figma/Kimi-class intent).
  3. Throughline Lab (optional) — Tonnetz funnel + throughline-ai.
  4. WebGrid / μgrad / kbatch / hexcast — gamify + live kinetics → hexterm.
  5. uvQbit + globe — quantum hub; Core Race — governance signal tree.
  6. Gridlock (here) — data handling & mapping.
  7. Other tabs (mueee, launcher, …) — same facet contract.
  8. Notes — column funnel, final handoff.

What it can do (target)

  • Show URL ↔ block ↔ timeline position links fed from HistorySearch and history graph state (via BroadcastChannel or shared export JSON).
  • Surface layout ↔ DOM region mappings from GridPad infer / facet exports next to the same URLs you found in Search.
  • Anchor QPU nodes (qbit-globe) to bookmarks, queries, or corpus categories—so map and reading list share one ID space.
  • Hold preflight / contrail snapshots for “why this page is wired this way” operator notes (DAC + prefix coverage).
  • Later: live data-flow hints (forms, fetch patterns) where HTML or extension capture allows—labeled as heuristic.

Open questions (discussion)

  • Single schema for a “throughline record” (url, sources[], gridpadDocRef?, historyBlockIds?, qpuFeatureId?) vs per-app IDs only.
  • Whether Gridlock stays read-mostly or gains editable “links” rows synced back to History.
  • How much to duplicate vs deep-link to iron-browser / plan-viewer for long-form provenance.
  • Privacy: only same-origin or user-pasted HTML for layout—no silent third-party fetch without explicit action.
